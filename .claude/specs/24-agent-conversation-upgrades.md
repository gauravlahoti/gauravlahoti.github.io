# Spec: Agent conversation upgrades — chips, citations, CTA, transparency

> **Depends on / extends:** spec **#20** (widget UI, FAB, panel, drag-to-dismiss), spec **#21** (ADK on Cloud Run, SSE protocol, `_stream_agent`, `instruction.py`, guardrails, rate limiter), spec **#23** (audit-log Worker endpoint and `agent_interactions` schema).

## Overview

Spec #21 shipped a working agent: it streams (SSE deltas already arrive frame-by-frame), it cites with a `Sources: [...]` footer, and it survives Cloud Run cold starts on the free tier. What it doesn't do is *feel* like a credentialed AI-native architect's agent. Three concrete gaps:

1. **No conversational momentum.** A response ends in a dead end. The model has session memory across the page-load; the UI doesn't show it. Visitors who'd happily ask a follow-up don't.
2. **Trust signals are flat.** `Sources: LinkedIn` as a footer reads as a disclaimer rather than evidence. There's no way for a visitor to follow a specific claim back to the post or section that supports it. And there's no "how does this thing work?" explainer on the widget itself.
3. **The off-ramp is wasted.** When a visitor asks something the agent shouldn't answer (salary, relocation, references, "is he open to Berlin?"), the polite decline currently terminates the conversation instead of routing to Topmate — exactly the moment a hot lead is asking the most personal questions.

This spec writes those three problems closed in one coherent change set, plus tightens the mobile sheet and adds a one-time scroll-triggered nudge. It deliberately does **not** rewrite Spec #21's transport (still SSE, still ADK runner, still in-memory sessions) and does **not** break the Spec #23 audit-log contract (the Worker endpoint stays compatible; new optional columns extend it).

A first-time visitor, after one question, should see: streaming text with a live caret, an inline `[1]` citation superscript on at least one factual claim, two or three follow-up chips that match the topic of their question, and — if they probe personal territory — a primary "Book a Topmate call →" button instead of a dead-end paragraph. If they scroll past the Career section without opening the widget, a single dismissible bubble appears once next to the FAB. The "Powered by ADK + Gemini" footer becomes clickable, opening a transparency modal that is itself a credential.

## Goals

1. **Follow-up chips** — render 2–3 dynamic, topic-aware chips after every successful answer, generated by the model in the same call (no extra latency, no second LLM call).
2. **Streaming feel** — keep the existing SSE delta protocol; add a typing caret and ensure the "Connecting…" loading indicator clears on the *first* delta, not on `done`.
3. **Inline citations** — replace the trailing `Sources: [...]` line with `[N]` superscript markers next to the phrase they support; clicking opens the source in a new tab; hover reveals a label.
4. **Graceful "I don't know" → Topmate CTA** — when the agent declines a personal/out-of-scope question, render a primary "Book a Topmate call →" button under the response.
5. **One-time scroll nudge** — when a desktop visitor scrolls past the Career section without opening the widget, show one dismissible bubble next to the FAB. Never re-show in the same browser session.
6. **Mobile sheet tightening** — default open height reduced from 75vh to 60vh; expanded max stays 92vh; z-index audit confirms no overlap with portrait, sticky bottom bar, or resume-gate modal.
7. **Transparency modal** — the "Powered by ADK + Gemini" footer becomes clickable and opens a `<dialog>` with a brief explainer (model, indexed snapshot, what it can't do).
8. **Eval gate stays green and gains regression coverage** for the new behaviours (chips present, citations present, personal-punt → topmate, off-topic-punt, meta-block well-formed).

## Non-goals

- **Live LinkedIn data retrieval.** Considered and rejected: there is no official LinkedIn MCP server; LinkedIn's official `/v2/posts` API requires Marketing Developer Platform partner approval and only returns the signed-in user's own data; every third-party scraping API (Bright Data, Netrows, Scrapingdog, LinkdAPI) explicitly violates LinkedIn's ToS — ProxyCurl, the previous market leader, was sued by LinkedIn in Jan 2026 and shut down on July 4, 2026. The frozen-per-deploy corpus from Spec #21 stays the source of truth; freshness comes from `make corpus` + redeploy + the existing `/add-post` slash command. Auto-refresh via a GitHub Actions cron is a viable follow-up but a separate spec.
- Changing the model (`gemini-2.5-flash` stays).
- Adding a second LLM call per turn — chips and citations come out of the same generation.
- Server-side citation grounding via Vertex AI grounding API. Eval rubric remains the hallucination gate.
- Persistent conversation history across page reloads (sessionId stays page-load-scoped, per Spec #21).
- Localisation / RTL layout for chip and CTA copy.
- A/B testing infrastructure for chip phrasing.
- Server-minted HS256 identity token (already noted as future hardening in Spec #23).

## Routes

**No new browser-facing route.** The widget continues to call `POST /api/agent-chat` and `GET /api/agent-chat/warm` on Cloud Run.

**SSE protocol extension** (non-breaking — older clients ignore unknown event keys):

| Event | Shape | When emitted |
|---|---|---|
| `delta` | `{"delta": "..."}` | unchanged; per text chunk |
| `citations` | `{"citations": [{"id": 1, "url": "https://...", "label": "LinkedIn — Multi-cloud"}]}` | once, after the model's text has finished, before `done` |
| `suggestions` | `{"suggestions": ["Show me the AWS projects", "What was the hardest migration?"]}` | once, after `citations`, before `done` |
| `cta` | `{"cta": "topmate" \| "linkedin"}` | optional, when the model emits a CTA marker |
| `done` | `{"done": true}` | unchanged; final |

**Worker endpoint** (`POST /api/agent-log`, Spec #23) extends to accept three optional fields: `citationsCount: int|null`, `suggestionsCount: int|null`, `cta: str|null`. Auth, CORS, rate-limit, and cron retention all unchanged. The browser still never talks to `/api/agent-log` directly.

## Architecture: how the model produces meta

`instruction.py` is updated so every reply ends with a single trailing block, fenced by literal sentinels:

```
…response text with [1] and [2] markers inline…
[[META]]
{"citations":[{"id":1,"url":"https://www.linkedin.com/posts/...","label":"LinkedIn — Multi-cloud"},
              {"id":2,"url":"https://github.com/...","label":"GitHub — repo"}],
 "suggestions":["Show me the AWS projects","What was the hardest migration?","Which post explains his stance?"],
 "cta":null}
[[/META]]
```

`_stream_agent` (in `app/api.py`):
- Buffers any cumulative text starting with `[[META]]` instead of streaming it as a `delta`.
- Closes on `[[/META]]` or end-of-stream, parses the JSON, and emits up to three structured SSE events (`citations`, `suggestions`, `cta`) before the final `done`.
- On any failure (sentinel missing, JSON malformed, output truncated mid-block), falls back silently to no chips / no citations / no CTA. The user-visible answer must never break because of meta.

This keeps the model contract a single text generation. No structured-output JSON-mode juggling, no second call, no extra latency.

### Sentinel injection hardening

The `[[META]]` / `[[/META]]` sentinels are short literal strings; a hostile visitor could embed them in a user message hoping the model echoes a forged meta block (e.g. forcing `cta: "topmate"` to bait a click). Two-layer defense:

1. **Pre-model strip** — in `app/guardrails.py`'s existing `before_model_callback`, strip `[[META]]` and `[[/META]]` substrings from the user message before it reaches the model. Visitors cannot smuggle the sentinels through the input channel.
2. **Last-wins parsing** — the parser anchors on the LAST `[[META]]` occurrence in the model's output (using `rfind`), not the first. If the model echoes a quoted attempt earlier in its body and emits its real meta block at the end (per the prompt template), the real block wins. Any earlier `[[META]]` substring in the body is treated as plain text and stays in the user-visible answer (cosmetic only — never parsed).

Together these mean: the only way to inject a meta payload is to compromise the model itself, at which point a forged citation URL still has to clear the server-side host allowlist before insert.

### Token budget

Spec #21 set `max_output_tokens = 1500`. The trailing meta block adds ~80–150 tokens to every reply, and on long answers (project deep-dives, multi-cloud histories) we risk truncating the meta block — or worse, truncating the user-visible body to make room. Bump to `1800` in `app/agent.py` (the only line that changes there) and add a sentence to `instruction.py` instructing the model to keep the meta block under 200 tokens (≤ 3 citations, ≤ 3 suggestions, terse labels). Truncated-meta is already a graceful no-op (§"Architecture"); the bump is purely to keep the user-visible body whole on long answers.

## Rules for implementation

### `portfolio-agent/app/instruction.py`

Surgical edit. Preserve persona / scope / URL allowlist text verbatim. Drop the existing requirement to end every factual reply with `Sources: [...]`. Replace with:

1. **Inline citation markers.** When stating a verifiable claim sourced from a tool result, insert `[1]`, `[2]`, or `[3]` immediately after the supporting phrase. Maximum 3 markers per reply. Never invent a citation. Never cite something that didn't come out of a tool call.
2. **Trailing meta block, mandatory on every reply** (including off-topic declines):
   ```
   [[META]]
   {"citations":[{"id":1,"url":"...","label":"..."}],
    "suggestions":["...","...","..."],
    "cta":null}
   [[/META]]
   ```
   - `citations[].url` MUST be on the existing host allowlist (`linkedin.com`, `github.com`, `topmate.io`, `gauravlahoti.github.io`).
   - `citations[].label` ≤ 80 chars; human-readable (e.g. `"LinkedIn — Multi-cloud post"`, `"GitHub — portfolio repo"`).
   - `suggestions` is a list of 2–3 strings, each ≤ 80 chars, phrased as a question a visitor might naturally ask next.
   - `cta` is `null`, `"topmate"`, or `"linkedin"`.
3. **Personal / out-of-knowledge questions** (salary, relocation, references, future intent, internal opinions, anything not in the corpus): the body of the response is a single brief sentence declining; `cta` MUST be `"topmate"`. Suggestions in this case should be questions the agent CAN answer (e.g. "What has he shipped on multi-cloud?").
4. **Off-topic questions** (weather, sports, politics, anything unrelated to Gaurav): brief decline; `cta` MAY be `"linkedin"` or `null` at the model's discretion.
5. **Guardrail short-circuits** (prompt-injection, too-long input) are emitted by callbacks BEFORE the model runs and do NOT include a meta block. The server detects these by the existing canned-prefix match (Spec #23) and emits zero structured events.
6. **Meta-block size cap.** Keep the meta block under 200 tokens: ≤ 3 citations, ≤ 3 suggestions, labels under 80 chars. This protects the user-visible body from getting truncated by `max_output_tokens` on long answers.

### `portfolio-agent/app/api.py`

1. Import the canned-prefix constants from `app.guardrails`:
   ```python
   from app.guardrails import INJECTION_REPLY_PREFIX, TOO_LONG_REPLY_PREFIX
   ```
   (Rename in `guardrails.py` from leading-underscore to public; the existing constants in `api.py` become re-exports for one release for backwards compat in the audit-log status detection.)

2. Add module-level constants:
   ```python
   _META_OPEN  = "[[META]]"
   _META_CLOSE = "[[/META]]"
   _ALLOWED_CTA = {"topmate", "linkedin"}
   _ALLOWED_CITE_HOSTS = {"linkedin.com", "www.linkedin.com",
                          "github.com", "topmate.io", "gauravlahoti.github.io"}
   ```

3. Extend `_stream_agent` state:
   ```python
   meta_buffer: list[str] = []
   meta_open  = False
   user_visible: list[str] = []   # the text we actually streamed (sans meta)
   citations_payload: list[dict] | None = None
   suggestions_payload: list[str] | None = None
   cta_payload: str | None = None
   ```

4. **Sentinel-safe streaming.** Replace today's "yield delta verbatim" with a buffer that holds back the trailing `len(_META_OPEN) - 1` characters until either:
   - the buffer doesn't end with a prefix of `[[META]]` (flush), OR
   - the full sentinel appears (set `meta_open = True`, stop forwarding to user, route remaining text to `meta_buffer`).
   
   Concretely: maintain a `pending` string. After each event's text arrives, while `pending` length > `len(_META_OPEN) - 1` and `pending` does not start with a strict prefix of `_META_OPEN`, flush one character. If a full match is found at any prefix, switch to meta mode at that index (do not flush the sentinel). The implementation is ~20 lines and is exercised by `test_meta_parser.py`.

5. **Meta close + parse.** When `_META_CLOSE` is observed in `meta_buffer` (or stream ends), use `rfind(_META_OPEN)` on the **cumulative response so far** to locate the LAST sentinel pair (last-wins — defends against the model echoing a quoted earlier sentinel from a hostile user message). Strip the open/close sentinels of that final pair, `json.loads` the inside, and validate:
   - `citations` is a list of dicts each with `id: int`, `url: str` (host in `_ALLOWED_CITE_HOSTS`), `label: str` (≤ 80 chars). Drop entries that fail; do not error the response.
   - `suggestions` is a list of 2–3 strings, each ≤ 80 chars. Trim to 3, drop empties.
   - `cta` is `None` or in `_ALLOWED_CTA`.
   
   On any exception or shape failure: log at `WARNING`, leave all three payloads as `None` / `[]` / `[]`. Do NOT raise.
   
   **Server is canonical for the citation host allowlist.** If a citation URL fails the allowlist check, drop the citation entry server-side; the frontend never sees it. This keeps a single source of truth.

6. **Emit structured events** before `_sse({"done": True})`:
   ```python
   if citations_payload:
       yield _sse({"citations": citations_payload})
   if suggestions_payload:
       yield _sse({"suggestions": suggestions_payload})
   if cta_payload:
       yield _sse({"cta": cta_payload})
   yield _sse({"done": True})
   ```

7. **Audit-log payload extension.** The existing `log_interaction` call passes `response = emitted[:16000]`. After this spec, `response` becomes the user-visible text (i.e. `"".join(user_visible)[:16000]` — explicitly excluding the meta block). Add three new fields:
   ```python
   "citationsCount":   len(citations_payload or []),
   "suggestionsCount": len(suggestions_payload or []),
   "cta":              cta_payload,
   ```
   These pass through Spec #23's existing payload validation (which rejects unknown keys today — see `handleAgentLog` in `backend/src/index.js` — so the Worker change in §"Worker" must ship before or with the agent change).

8. **Guardrail short-circuit detection** stays the same (canned-prefix match on `emitted[:200]`); when fired, `citations_payload / suggestions_payload / cta_payload` stay empty, and no structured events are emitted.

1. **Top-of-file feature flags** (single-flip rollback):


2. **SSE parser extension** (in the existing `streamAgent` block). Today the JSON event has two known shapes (`{delta}`, `{done}`). Extend the dispatch:
   ```js
   if (typeof evt.delta === "string") onDelta(evt.delta);
   else if (evt.citations)            onCitations(evt.citations);
   else if (evt.suggestions)          onSuggestions(evt.suggestions);
   else if (evt.cta)                  onCta(evt.cta);
   else if (evt.done === true)        onDone();
   ```

3. **Citation rendering — runs once, after `done`.** Extend `renderTextWithLinks(text, citations)` to accept an optional `citations` map keyed by id. After the existing URL-linking pass, replace `[1]`, `[2]`, `[3]` patterns:
   ```js
   text = text.replace(/\[(\d)\]/g, (m, n) => {
       const c = citations?.[Number(n)];
       if (!c) return m; // server already enforced the allowlist; if we got it, trust it
       return `<sup class="agent-cite"><a href="${escapeAttr(c.url)}" target="_blank" rel="noopener" title="${escapeAttr(c.label)}" data-cite-id="${n}">[${n}]</a></sup>`;
   });
   ```
   **Order of operations** (locks down the re-render bug surface):
   - During streaming, `[N]` markers render as **plain text** inside the in-progress message — no superscript yet, typing caret active.
   - The `citations` SSE event arrives before `done`; on receipt, store `pendingCitations = byId(citations)` but do NOT re-render yet.
   - On `done`: remove the typing caret first, THEN re-run `renderTextWithLinks(finalText, pendingCitations)` once over the message's text node only (not the whole transcript), THEN render the chips and CTA.
   - This guarantees: (a) the caret is never wiped mid-stream by a citations re-render, (b) no in-flight delta paints into a stale DOM node, (c) URL linkification runs exactly once on the final text — no double-linkification.
   
   **Frontend trusts the server's allowlist decision.** The agent is canonical for the citation host allowlist; the frontend defensively `escapeAttr`s the URL into the `href` attribute and renders. If a citation came through, the server already admitted it.

4. **Follow-up chips**. After `done`, render a row of `<button class="agent-suggestion-chip">…</button>` under the latest assistant message:
   ```js
   const row = document.createElement("div");
   row.className = "agent-suggestions";
   suggestions.forEach(s => {
       const b = document.createElement("button");
       b.className = "agent-suggestion-chip";
       b.type = "button";
       b.textContent = s;
       b.addEventListener("click", () => sendMessage(s));
       row.appendChild(b);
   });
   lastAssistantMsg.appendChild(row);
   ```
   When a chip is clicked OR the user types another message, the chip row is removed from the previous assistant message (chips belong only to the most recent answer).

5. **CTA block**. After `done`, if a CTA was emitted, look up label + URL from `profile.agentCopy.cta` (no hardcoded strings — copy lives in data, not code):
   ```js
   const entry = profile.agentCopy?.cta?.[cta]; // { label, url }
   if (entry?.url) appendCtaButton(lastAssistantMsg, entry.label, entry.url);
   ```
   The button uses `.agent-cta-action` styling (primary accent matching existing site CTAs) and opens in a new tab. See §"`assets/js/data/profile.json`" for the copy block.

6. **Typing caret**. While a response is streaming, append a `<span class="agent-cursor"></span>` to the in-progress assistant message text node. Remove it on `done`. Visible block, blinks at 1Hz, no animation when `prefers-reduced-motion`.

7. **Loading-stage dismissal fix**. In the existing `startLoadingStages()` flow, today the "Connecting…" indicator clears at the wrong time on slow first-token starts. Two-line fix: call the cancel function from inside the first `onDelta` invocation (not just at `done`). Verify with a chrome devtools network throttle ("Slow 4G") that the indicator clears as soon as text starts to render.

8. **Scroll nudge** (desktop only). Module-level state — declared at the top of `initAgentWidget`:
   ```js
   let panelEverOpened = false; // flipped to true the first time the panel opens this page-load
   ```
   The existing panel-open handler must `panelEverOpened = true;` on first open. Then:
   ```js
   if (FEATURES.scrollNudge && matchMedia("(min-width: 768px)").matches) {
       const career = document.querySelector("#career, [data-section='career']");
       const NUDGE_KEY = "agent_nudge_v1";
       if (career && sessionStorage.getItem(NUDGE_KEY) !== "shown") {
           const io = new IntersectionObserver((entries) => {
               for (const e of entries) {
                   if (e.isIntersecting && !panelEverOpened) {
                       showNudge();
                       sessionStorage.setItem(NUDGE_KEY, "shown");
                       io.disconnect();        // fire once; never observe again
                   }
               }
           }, { threshold: 0.4 });
           io.observe(career);
       }
   }
   ```
   `showNudge()` renders a `<div class="agent-nudge">` next to the FAB with copy from `profile.agentCopy.nudge.label` and a close `×`. Click → opens panel + auto-sends `profile.agentCopy.nudge.prompt`. Auto-dismiss after 8s. Manual dismiss (`×`) also sets the sessionStorage key (already done above) and calls `io.disconnect()` defensively if not already disconnected. Mobile is exempt — the sticky bottom bar already handles entry there.

9. **Transparency modal**. Convert the footer text "Powered by ADK + Gemini" into a `<button class="agent-explainer-trigger">`. On click, open a native `<dialog class="agent-explainer-dialog">` rendering `profile.agentExplainer.body` (HTML-escaped paragraphs) plus a footer link to `profile.agentExplainer.repoUrl`. Close on `Escape`, click-outside-content, or an explicit `×`. Mark `aria-modal="true"`. Respects `prefers-reduced-motion` (no slide-in).

10. **No identity-payload change.** The `identity` claim from Spec #23 still flows; nothing here touches it.

11. **Error states.** Three failure modes during a turn:
    - **Network failure mid-stream** (fetch rejects, or the SSE connection drops between deltas): remove the typing caret immediately, keep whatever partial text streamed so far in the transcript, and append an inline `Connection dropped — retry?` button styled as `.agent-retry-inline`. Click → re-sends the same user message (the existing `messages` array is intact; sessionId is the same). Do NOT remove the partial text on retry — append a fresh assistant message instead.
    - **Server 5xx / 4xx (non-429)** before any delta arrives: no transcript pollution. Render the existing fallback line ("server error, try again or reach Gaurav on LinkedIn") in the loading-stage area and clear it on the next user input. No retry button (the model didn't run; user-driven retry is fine).
    - **429 rate-limited**: existing behaviour — render the canned cap message, do NOT show retry.
    
    In all three cases: caret is removed, no `done` event is required to clean up state, and `panelEverOpened` stays `true` (rate-limited / errored visitors still saw the panel).
